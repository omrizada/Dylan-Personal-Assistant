# Frameworks Library

Shared reference for all analytical, strategic, and coaching frameworks. Agents and skills reference this file instead of defining frameworks inline.

See also [[analyst]], [[strategist]], [[critic]], [[Dashboard]].

## Analytical Frameworks

### SWOT Analysis
**Use when**: Evaluating a position, product, or initiative holistically
**Process**: Map Strengths, Weaknesses, Opportunities, Threats → cross-reference for strategic options
**Output**: 2x2 matrix + strategic implications

### Root Cause Analysis (5 Whys)
**Use when**: Diagnosing why something failed or underperformed
**Process**: Start with the problem → ask "Why?" 5 times → trace to root cause
**Output**: Causal chain + root cause + prevention action

### Pareto Analysis (80/20)
**Use when**: Identifying where effort will have disproportionate impact
**Process**: List all factors → rank by impact → identify the 20% driving 80% of results
**Output**: Ranked list with cumulative impact

### Cohort Analysis
**Use when**: Comparing groups over time or across segments
**Process**: Define cohorts → measure same metric across all → identify divergence
**Output**: Comparison table + key divergence points

### Gap Analysis
**Use when**: Understanding distance between current state and target state
**Process**: Define current state → define target → map the gap → prioritize closing actions
**Output**: Gap table with severity and closing actions

### Trend Analysis
**Use when**: Understanding direction of change over time
**Process**: Collect time-series data → identify patterns → project trajectory → assess implications
**Output**: Trend summary with trajectory and inflection points

## Strategic Frameworks

### OKR Development
**Use when**: Setting and tracking quarterly objectives
**Process**: Define 3-5 Objectives (qualitative) → 3-5 Key Results each (quantitative, measurable) → weekly progress check
**Output**: OKR table with targets, current, status (on-track/at-risk/off-track)

### First Principles Thinking
**Use when**: Approaching a problem without assumptions
**Process**: Break down to fundamental truths → rebuild reasoning from scratch → challenge every assumption
**Output**: Fundamental truths + rebuilt logic + novel conclusions

### Scenario Planning
**Use when**: Preparing for multiple possible futures
**Process**: Identify key uncertainties → create 3-5 plausible scenarios → model implications → develop contingencies
**Output**: Scenario matrix with triggers, implications, and contingency plans

### Porter's Five Forces
**Use when**: Analyzing competitive dynamics in a market
**Process**: Map supplier power, buyer power, competitive rivalry, threat of substitutes, threat of new entrants
**Output**: Force assessment with strategic implications

### Decision Matrix
**Use when**: Choosing between multiple options with multiple criteria
**Process**: List options → define criteria with weights → score each option → calculate weighted total
**Output**: Scored matrix with recommendation

### Eisenhower Matrix
**Use when**: Prioritizing tasks by urgency and importance
**Process**: Classify each task into: Do (urgent+important), Schedule (important, not urgent), Delegate (urgent, not important), Eliminate (neither)
**Output**: 2x2 classification with action for each task

### ICE Scoring
**Use when**: Quick prioritization of ideas or features
**Process**: Score each item: Impact (1-10) × Confidence (1-10) × Ease (1-10) → rank by total
**Output**: Ranked list with ICE scores

### RICE Scoring
**Use when**: Product feature prioritization
**Process**: Reach × Impact × Confidence ÷ Effort → rank by score
**Output**: Ranked feature list with RICE scores

## Critique Frameworks

### Pre-Mortem Analysis
**Use when**: Stress-testing a plan before execution
**Process**: Assume the project has failed → work backward to identify all possible failure modes → rank by likelihood × impact → create prevention plans
**Output**: Failure mode table + prevention strategies

### Red Team Analysis
**Use when**: Finding vulnerabilities in a strategy or system
**Process**: Adopt adversary mindset → attack the plan from every angle → document exploitable weaknesses → propose defenses
**Output**: Vulnerability list with severity and mitigation

### Assumption Mapping
**Use when**: Surfacing hidden assumptions in a plan
**Process**: List all assumptions (explicit and implicit) → classify confidence (known/believed/hoped) → test highest-risk assumptions first
**Output**: Assumption table with confidence and test plan

### Stakeholder Impact Analysis
**Use when**: Understanding how a decision affects different groups
**Process**: Map all stakeholders → assess impact (positive/negative/neutral) per stakeholder → identify winners, losers, and risks
**Output**: Stakeholder impact matrix

## Coaching Frameworks

### SBI (Situation-Behavior-Impact)
**Use when**: Giving specific, behavioral feedback
**Process**: Describe the Situation → the specific Behavior observed → the Impact it had
**Output**: Structured feedback statement

### GROW (Goal-Reality-Options-Will)
**Use when**: Coaching conversations and problem-solving
**Process**: Define the Goal → assess current Reality → explore Options → commit to actions (Will)
**Output**: Action plan with commitments

### NVC (Nonviolent Communication)
**Use when**: Conflict resolution, difficult conversations
**Process**: Observe without judgment → express Feelings → identify Needs → make specific Requests
**Output**: Communication script using OFNR format

## Negotiation Frameworks

### BATNA (Best Alternative to Negotiated Agreement)
**Use when**: Preparing for any negotiation
**Process**: Identify your walkaway option → assess its value → use as leverage floor
**Output**: BATNA statement with value assessment

### ZOPA (Zone of Possible Agreement)
**Use when**: Finding where deal is possible
**Process**: Identify your range (reservation price to aspiration) → estimate counterparty's range → find overlap
**Output**: ZOPA diagram with recommended opening position
